# ClickFlow

## Описание

**ClickFlow** — минималистичный кроссплатформенный кликер и визуальный автоматизатор для быстрых сценариев.

Приложение позволяет запускать автоматические клики по заданным координатам, создавать сценарии автоматизации и управлять ими через простой и понятный интерфейс.

## Главная идея

- **Простое главное меню**: выбрать сценарий → нажать Start → при необходимости нажать Stop.
- **Расширенный режим**: все сложные функции (OCR, поиск по изображению, детальные настройки) спрятаны от основного интерфейса.
- **Быстрый запуск**: пользователь открывает приложение и сразу видит текущий сценарий и кнопку запуска.

## Текущий статус

Проект находится на раннем этапе разработки (Шаг 2 из запланированных).

Реальный автокликер, OCR и распознавание изображений пока не реализованы. Сейчас готова архитектурная основа, интерфейс и система логирования.

## Как запустить

```bash
npm install
npm start
```

Также доступна команда `npm run dev` (эквивалентна `npm start`).

## Структура проекта

```
clickflow/
  package.json           — конфигурация проекта, скрипты, зависимости
  main.js                — главный процесс Electron (окно, lifecycle)
  preload.js             — безопасный мост в renderer через contextBridge
  README.md              — документация проекта
  src/
    index.html           — главный HTML (главный экран + расширенный режим)
    styles.css           — стили приложения (CSS-переменные, минимализм)
    renderer.js          — логика отрисовки и обработки событий
    app-state.js         — модуль состояния приложения
    logger.js            — модуль логирования событий
    scenario-manager.js  — менеджер сценариев (базовая структура)
```

## История разработки

### Шаг 1 — Базовый Electron-проект

Что было сделано:
- создана базовая структура проекта (package.json, main.js, preload.js, src/);
- добавлено главное окно Electron (1000×700, minWidth/minHeight);
- добавлен минималистичный главный экран с карточкой статуса;
- добавлены кнопки Start и Stop с визуальной сменой статуса;
- подготовлен preload.js с contextBridge;
- contextIsolation включён, nodeIntegration отключён.

### Шаг 2 — Состояние, логи и база сценариев

Что изменилось:
- состояние приложения вынесено в отдельный модуль (`app-state.js`);
- добавлен модуль логирования (`logger.js`) с типами info/success/warning/error;
- добавлен менеджер сценариев (`scenario-manager.js`) с базовым сценарием;
- на главном экране появился блок «Последние события» с 5 последними логами;
- добавлена заглушка расширенного режима с кнопкой «Назад»;
- renderer.js переписан: использует модули, управляет видами (main/advanced);
- создан README.md с описанием проекта и историей шагов.

## Следующий шаг

На **Шаге 3** планируется:
- полноценная система сценариев (создание, редактирование, удаление);

- сохранение сценариев в памяти или JSON-файл.
