# ClickFlow

## Описание

**ClickFlow** — минималистичный кроссплатформенный кликер и визуальный автоматизатор для быстрых сценариев.

Приложение позволяет запускать автоматические клики по заданным координатам, создавать сценарии автоматизации и управлять ими через простой и понятный интерфейс.

## Главная идея

- **Простое главное меню**: выбрать сценарий → нажать Start → при необходимости нажать Stop.
- **Расширенный режим**: все сложные функции (OCR, поиск по изображению, детальные настройки) спрятаны от основного интерфейса.
- **Быстрый запуск**: пользователь открывает приложение и сразу видит текущий сценарий и кнопку запуска.

## Текущий статус

Проект находится на раннем этапе разработки (Шаг 4 из запланированных).

Реализован безопасный движок выполнения сценариев с имитацией кликов, прогрессом и остановкой. Реальных системных кликов пока намеренно нет — сейчас проверяется архитектура, UI прогресса и управление выполнением.

## Как запустить

```bash
npm install
npm start
```

Также доступна команда `npm run dev` (эквивалентна `npm start`).

## Структура проекта

```
clickflow/
  package.json           — конфигурация проекта, скрипты, зависимости
  main.js                — главный процесс Electron (окно, IPC, работа с файлами)
  preload.js             — безопасный мост в renderer через contextBridge
  README.md              — документация проекта
  src/
    index.html           — HTML с 4 видами: main, scenarios, scenarioForm, advanced
    styles.css           — стили приложения (CSS-переменные, минимализм)
    renderer.js          — логика интерфейса, управление видами и событиями
    app-state.js         — модуль состояния приложения (включая execution)
    logger.js            — модуль логирования событий
    scenario-manager.js  — менеджер сценариев (CRUD, валидация, сохранение)
    click-engine.js      — движок выполнения сценариев (имитация, прогресс, остановка)
```

## Сценарии

Сценарии сохраняются в `userData/scenarios.json` (автоматическая директория Electron).

Формат сценария:
- **id** — уникальный идентификатор
- **name** — название
- **type** — тип (пока только `simple_click`)
- **description** — описание
- **settings** — координаты (x, y), интервал (мс), повторы, кнопка мыши
- **meta** — дата создания, обновления, флаг базового сценария

Базовый сценарий «Быстрый кликер» всегда присутствует и не может быть удалён.

- contextIsolation включён, nodeIntegration отключён.

### Шаг 2 — Состояние, логи и база сценариев

Что изменилось:
- состояние приложения вынесено в отдельный модуль (`app-state.js`);
- добавлен модуль логирования (`logger.js`) с типами info/success/warning/error;
- добавлен менеджер сценариев (`scenario-manager.js`) с базовым сценарием;
- на главном экране появился блок «Последние события» с 5 последними логами;
- добавлена заглушка расширенного режима с кнопкой «Назад»;
- renderer.js переписан: использует модули, управляет видами (main/advanced);
- создан README.md с описанием проекта и историей шагов.

### Шаг 3 — Управление сценариями

Что изменилось:
- добавлен экран списка сценариев с карточками;
- добавлен выбор активного сценария из списка;
- добавлена форма создания нового сценария (название, описание, X, Y, интервал, повторы, кнопка мыши);
- добавлена форма редактирования существующего сценария;
- добавлено удаление пользовательских сценариев с подтверждением;
- базовый сценарий «Быстрый кликер» защищён от удаления;
- добавлена валидация полей формы с отображением ошибок;
- добавлено сохранение сценариев через IPC в `userData/scenarios.json`;
- preload.js расширен безопасным API для сценариев (load/save/reset);
- main.js получил IPC-обработчики для чтения/записи/сброса файла сценариев;
- app-state.js расширен полями для формы (scenarioFormMode, editingScenarioId);
- scenario-manager.js полностью переписан с CRUD-операциями и валидацией;
- DOM-элементы создаются через createElement/textContent (защита от XSS);
- README.md обновлён.

### Шаг 4 — Безопасный движок выполнения сценариев

Что изменилось:
- добавлен `src/click-engine.js` — движок выполнения с имитацией кликов;
- реализована имитация выполнения сценария `simple_click` (без реальных системных кликов);
- добавлен визуальный прогресс выполнения (progress bar, счётчик, процент);
- добавлено отображение последнего действия (координаты, кнопка мыши);
- добавлена остановка сценария во время выполнения через кнопку Stop;
- добавлена защита от повторного запуска (Start блокируется во время выполнения);
- кнопки Start/Stop динамически включаются/отключаются по состоянию;
- логирование хода выполнения (первые 3 действия, каждое 10-е, последнее);
- `app-state.js` расширен объектом `execution` (progressCurrent, progressTotal, progressPercent, lastAction, startedAt, finishedAt);
- `renderer.js` переписан: использует click-engine через callbacks;
- архитектура подготовлена к замене `simulateClick()` на реальный системный клик;
- реальные системные клики намеренно не реализованы на этом шаге;
- README.md обновлён.
