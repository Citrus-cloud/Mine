# ClickFlow

## Описание

**ClickFlow** — минималистичный кроссплатформенный кликер и визуальный автоматизатор для быстрых сценариев.

Приложение позволяет запускать автоматические клики по заданным координатам, создавать сценарии автоматизации и управлять ими через простой и понятный интерфейс.

## Главная идея

- **Простое главное меню**: выбрать сценарий → нажать Start → при необходимости нажать Stop.
- **Расширенный режим**: все сложные функции (OCR, поиск по изображению, детальные настройки) спрятаны от основного интерфейса.
- **Быстрый запуск**: пользователь открывает приложение и сразу видит текущий сценарий и кнопку запуска.
- **Два языка интерфейса**: русский и английский, переключаются в настройках.

## Текущий статус

Проект находится на раннем этапе разработки (Шаг 5 из запланированных).

Реализованы: система сценариев, безопасный движок выполнения с имитацией кликов, настройки приложения, локализация RU/EN, горячие клавиши, аварийная остановка и безопасные лимиты. Реальных системных кликов пока намеренно нет.

## Как запустить

```bash
npm install
npm start
```

Также доступна команда `npm run dev` (эквивалентна `npm start`).

    renderer.js            — логика интерфейса, i18n, hotkeys, управление видами
    app-state.js           — модуль состояния (execution, settings)
    logger.js              — модуль логирования событий
    scenario-manager.js    — менеджер сценариев (CRUD, валидация, сохранение)

## Горячие клавиши

| Комбинация | Действие |
|---|---|
| Ctrl+Alt+S | Запуск сценария |
| Ctrl+Alt+X | Остановка сценария |
| Escape | Аварийная остановка |

## Безопасность

- **Безопасный режим** — всегда включён, ограничивает параметры выполнения.
- **Минимальный интервал** — по умолчанию 50 мс (настраивается).
- **Максимум повторов** — по умолчанию 100 000 (настраивается).
- **Аварийная остановка** — Escape мгновенно прекращает выполнение.
- Реальных системных кликов пока нет — выполнение является имитацией.

## Локализация

Поддерживаемые языки: русский (по умолчанию), английский. Переключение в настройках. Все пользовательские тексты вынесены в систему переводов (data-i18n).
